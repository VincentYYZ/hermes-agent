# Hermes Agent — 架构与功能模块分析

## 项目定位

**Hermes Agent** 是由 [Nous Research](https://nousresearch.com) 构建的自改进 AI Agent 框架，核心特性：
- 支持 200+ LLM 模型（OpenRouter、Anthropic、OpenAI 等）
- 内置"学习闭环"：技能创建、记忆持久化、跨会话历史搜索
- 多平台接入：CLI、Telegram、Discord、Slack、WhatsApp、Signal 等 17+ 平台
- 六种终端后端：本地、Docker、SSH、Daytona、Modal、Singularity

## 核心模块详解

### 1. Agent 主核 — [run_agent.py](cci:7://file:///mnt/Uptec777/ai-agent-project/hermes-agent/run_agent.py:0:0-0:0) (535KB)
- **`AIAgent`** 类：整个系统的调度中心，同步对话循环
- 核心方法：`chat()` (简单接口) / `run_conversation()` (完整接口，返回 dict)
- 消息格式：OpenAI 标准格式 (`system/user/assistant/tool`)
- 内置 budget 管理 (`iteration_budget`)、`max_iterations=90`

## 依赖调用链（关键）

```
tools/registry.py  ← 无依赖，被所有工具文件导入
       ↑
tools/*.py         ← 各自调用 registry.register() 完成注册
       ↑
model_tools.py     ← _discover_tools() 触发所有工具导入
       ↑
run_agent.py / cli.py / batch_runner.py / environments/
```

## 配置体系

- **`~/.hermes/config.yaml`** — 用户设置（`DEFAULT_CONFIG` 版本 5）
- **`~/.hermes/.env`** — API Keys（`OPTIONAL_ENV_VARS` 分 provider/tool/messaging/setting 类）
- 两套独立加载器：CLI 用 `load_cli_config()`，工具管理用 `load_config()`，Gateway 直接 YAML 加载
